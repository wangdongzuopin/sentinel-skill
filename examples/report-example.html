Findings

CRITICAL  FIND-001: SQL Injection

CWE: CWE-89  |  CVSS: 9.8  |  Location: server.js:22

Description: User-supplied input from the username query parameter is directly concatenated into a SQL query without parameterization.

Impact: Attacker can extract, modify, or delete all database data including hardcoded credentials.

Evidence:

const sql = `SELECT * FROM users WHERE username = '${username}'`;
// Attack: ?username=' OR '1'='1 → returns all users
Recommendation: Use parameterized queries: db.all("SELECT * FROM users WHERE username = ?", [username], ...)

CRITICAL  FIND-002: Command Injection

CWE: CWE-78  |  CVSS: 9.8  |  Location: server.js:35

Description: The host parameter is passed directly to execSync() without validation.

Impact: Remote Code Execution — attacker gains full control of the server OS.

Evidence:

execSync(`ping -n 1 ${host}`, ...);
// Attack: {"host": "127.0.0.1; cat /etc/passwd"}
Recommendation: Use execFile with strict input validation: const validHost = /^[a-zA-Z0-9.-]+$/.test(host) ? host : null

CRITICAL  FIND-003: Exposure of Sensitive Information

CWE: CWE-798  |  CVSS: 9.1  |  Location: server.js:63-67

Description: The /api/config endpoint exposes database passwords, AWS keys, and JWT secrets in plain text.

Impact: Full database compromise, AWS account takeover, authentication bypass.

Evidence:

res.json({
  DATABASE_URL: 'mysql://admin:SuperSecret123@localhost:3306/prod',
  AWS_ACCESS_KEY: 'AKIAIOSFODNN7EXAMPLE',
  JWT_SECRET: 'my-super-secret-jwt-key-12345',
  DEBUG: 'true'
});
Recommendation: Remove this endpoint. Store secrets in environment variables only.

HIGH  FIND-004: Cross-Site Scripting (XSS)

CWE: CWE-79  |  CVSS: 8.2  |  Location: server.js:46

Description: User-supplied Markdown content is parsed and rendered as HTML without sanitization via marked.parse().

Impact: Session hijacking, credential theft, defacement.

Evidence:

const html = marked.parse(content || '');
res.send(`<html><body>${html}</body></html>`);
// Attack: ?content=<script>alert(document.cookie)</script>
Recommendation: Sanitize with DOMPurify: DOMPurify.sanitize(marked.parse(content))

HIGH  FIND-005: Path Traversal

CWE: CWE-22  |  CVSS: 7.5  |  Location: server.js:52

Description: User-supplied filename parameter is directly concatenated to the file path without validation.

Impact: Read arbitrary files on the server.

Evidence:

const filepath = path.join(__dirname, 'uploads', filename);
// Attack: ?filename=../../server.js
Recommendation: Validate and resolve path: const safePath = path.resolve(...); if (!safePath.startsWith(baseDir)) return 403

HIGH  FIND-006: Insecure Direct Object Reference (IDOR)

CWE: CWE-639  |  CVSS: 7.1  |  Location: server.js:74

Description: /api/user/:id returns user data for any ID without verifying requester's authorization.

Impact: Any user can view, modify, or delete other users' data by changing the ID.

Evidence:

db.get(`SELECT * FROM users WHERE id = ${id}`, ...);
// User 2 can access User 1's data by GET /api/user/1
Recommendation: Add authorization check: if (requestingUser.id !== targetId && requestingUser.role !== 'admin') return 403
